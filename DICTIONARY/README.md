# 지식 사전 작성 가이드

이 문서는 지식 사전(Obsidian 클론)을 처음 사용하는 사람을 위한 가이드라인입니다. 마크다운과 양방향 링크를 활용하여 나만의 지식 네트워크를 구축하는 방법을 설명합니다.

---

## 1. 핵심 개념: 양방향 링크 (Wiki-link)

지식 사전의 가장 중요한 기능은 문서와 문서를 유기적으로 연결하는 것입니다. 

* **링크 만들기**: 본문 작성 중 다른 노트로 연결하고 싶다면 `[[노트 이름]]` 형태로 작성합니다.
  * 예시: React에 대해 쓰던 중 `[[자바스크립트]]`를 입력하면 자바스크립트 노트로 바로 가는 링크가 만들어집니다.
* **링크를 통해 새 노트 만들기**: 아직 존재하지 않는 노트를 `[[새로운 주제]]`로 미리 적어두고, 미리보기(Preview) 모드에서 이를 클릭하면 해당 이름의 새 마크다운 파일이 표준 템플릿과 함께 자동으로 생성됩니다.

---

## 2. 문서의 머리말 (YAML Frontmatter) 작성법

모든 문서의 맨 위에는 `---`로 둘러싸인 메타데이터 영역이 있습니다. 이 정보는 향후 구현될 그래프 뷰에서 지식을 분류하고 검색할 때 중요하게 사용됩니다.

```yaml
---
title: "노트 제목"
created: 2026-06-03 18:00:00
updated: 2026-06-03 18:00:00
type: "concept"
status: "inbox"
tags:
  - "inbox"
aliases: []
---
```

* **title**: 노트의 제목입니다. 새 노트를 만들 때 자동으로 입력됩니다.
* **created / updated**: 노트가 생성된 시간과 마지막으로 수정된 시간입니다. 파일 저장 시 참고용으로 활용됩니다.
* **type**: 노트의 성격을 분류합니다. 아래 키워드 중 하나를 적절히 선택하여 입력하세요.
  * `concept`: 용어 정의나 핵심 이론 등 순수 개념 노트
  * `project`: 내가 현재 진행 중인 작업이나 개발 프로젝트
  * `daily`: 일상 기록이나 일간 업무 일지
  * `resource`: 웹 서핑 자료, 스크랩, 책 요약 등 외부 정보
* **status**: 지식의 성숙도를 나타냅니다.
  * `inbox`: 방금 생성하여 내용이 거의 없는 임시 보관 상태 (새 노트 기본값)
  * `seed`: 지속적으로 살을 붙이고 공부하며 가꾸는 중인 상태
  * `evergreen`: 완전히 이해하고 정리하여 더 이상 크게 바뀔 일이 없는 지식 상태
* **tags**: 문서의 카테고리 태그입니다. 여러 개를 추가할 수 있습니다.
* **aliases**: 이 노트를 부르는 다른 이름들(동의어)입니다.
  * 예시: title이 "React"인 경우 `aliases: [리액트, ReactJS]`로 적어두면, 다른 문서에서 `[[리액트]]`나 `[[ReactJS]]`로 적어도 이 문서로 연결됩니다.

---

## 3. 표준 문서 구조

처음 노트를 만들면 아래와 같은 3단계 뼈대가 제공됩니다. 이 구성을 채워 넣으며 작성하면 체계적인 정리가 가능합니다.

### 개요
이 문서가 무엇인지에 대해 직관적으로 이해할 수 있도록 한 줄 내외로 요약이나 핵심 정의를 작성합니다.

### 연관 노트
* **상위 주제**: 이 노트보다 한 단계 넓은 범주의 주제 노트를 링크합니다. (예: React Hook 노트의 상위 주제는 `[[React]]`)
* **관련 노트**: 함께 읽으면 유용한 형제 노트를 링크합니다. (예: `[[React Component]]`)

### 본문
실제 설명과 개념을 마크다운 문법을 활용하여 자유롭게 작성합니다. 중요한 개념어나 용어는 계속해서 `[[링크]]`를 걸어 지식의 그물을 넓혀 나가세요.
