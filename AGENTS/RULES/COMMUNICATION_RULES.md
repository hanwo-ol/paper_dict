# Communication Rules (대화 규칙)

## 1. 기본 원칙 (General Principles)
- **언어:** 모든 답변은 한국어로 간결하고 명확하게 작성한다.
- **명확성:** 불확실한 요구사항이나 중요한 설계 결정은 임의로 판단하지 않고 반드시 사용자에게 질문하여 확답을 받는다.
- **주도성:** 코드를 제안하거나 변경하기 전, '무엇을, 왜' 변경하는지 그 의도와 설계 방향을 먼저 설명한다.

## 2. Vibe Coding 특화 규칙
- **빠른 피드백:** 장황한 설명보다는 핵심적인 내용 전달과 실제로 동작하는 코드(또는 데모)를 우선시한다.
- **단계별 진행:** 한 번에 너무 많은 코드를 작성하지 않고, 작은 마일스톤 단위로 구현한 뒤 사용자의 승인과 피드백을 거쳐 다음 단계로 넘어간다.
