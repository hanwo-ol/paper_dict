# Coding Rules (코딩 규칙)

## 1. 기술 스택 및 아키텍처
- **핵심 스택:** React, TypeScript, Vite
- **디자인/스타일링:** 미니멀 다크 모드 (옵시디언 스타일 기본 탑재). 커스텀 CSS 또는 Tailwind CSS를 일관성 있게 사용한다.
- **데이터 관리 방식:** 로컬 우선(Local-first). 브라우저의 File System Access API를 이용하여 로컬 폴더 내의 `.md` 파일을 직접 읽고 쓰며 별도의 백엔드를 구축하지 않는다.

## 2. 코드 품질 및 Vibe Coding 원칙
- **모듈화 구조:** UI 화면(에디터, 사이드바 트리, 그래프 뷰)과 비즈니스 로직(파일 입출력, 위키 링크 파싱)은 철저히 분리된 컴포넌트로 분할하여 작업한다.
- **상태 관리:** 파일 상태나 선택된 문서 정보 등 전역 관리가 필요한 상태는 최소화하되 명확하게 관리(Context API 등)하며, 단방향 데이터 흐름을 지향한다.
- **방어적 코딩 (안정성 우선):** Vibe Coding 중 새로운 기능을 추가할 때 기존 핵심 로직(특히 마크다운 파싱 및 저장 로직)이 손상되지 않도록 항상 주의하여 변경한다.
