# Documentation Rules (문서 작성 규칙)

## 1. 아티팩트 및 문서화 기준
- 주요 기획서, 계획서(Implementation Plan), 할 일 목록(Task)은 항상 지정된 아티팩트 폴더(Artifacts)에 마크다운(`.md`)으로 작성하여 관리한다.
- 문서 내에서 특정 파일 경로나 코드의 함수명/클래스명을 언급할 때는 반드시 클릭하여 이동할 수 있는 파일 링크 형식(`[파일명](file:///절대/경로)`)을 사용한다.

## 2. 체크리스트 및 상태 관리
- 작업 진행 상황이나 검증(Verification)이 필요한 사항은 마크다운 체크리스트(`- [ ]`, `- [x]`)를 적극 활용하여 가시성을 높인다.
- 단계별 진행 상태를 Task 문서에 실시간으로 반영한다.

## 3. 코드 주석 (Code Comments)
- 주요 UI 컴포넌트(파일 트리, 에디터 등)나 복잡한 비즈니스 로직(링크 파싱, 파일 입출력) 위에는 JSDoc 스타일 또는 명확한 형태의 주석을 추가하여 향후 유지보수를 돕는다.
- 코드를 단순히 설명하는 주석보다는, '왜(Why)' 이렇게 구현했는지를 설명하는 주석을 지향한다.
